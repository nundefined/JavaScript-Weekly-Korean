# 읽을 거리

## [Native Speed on the Web with JavaScript &amp; asm.js](http://kripken.github.io/mloc_emscripten_talk/sloop.html?#/)
An illuminating slidedeck from Mozilla’s Alon Zakai, the main developer of Emscripten, that digs into the tricks and techniques behind getting native-like performance for transpiled native code on the Web.

## [JavaScript Modules, AMD, and The Road Ahead](http://yahooeng.tumblr.com/post/62383009835/javascript-modules-amd-and-the-road-ahead)

## [Making A Collaborative Piano Using Functional Reactive Programming](http://open.bekk.no/making-a-collaborative-piano-using-functional-reactive-programming-frp#?)
Explains *Functional Reactive Programming* (FRP) via an example. FRP is a declarative programming paradigm combining functional and reactive programming that works with *signals*, values that change over time and stay connected (similar to data binding). FRP works well for implementing graphical user interfaces.

## [How Complex are TodoMVC Implementations](http://blog.coderstats.net/todomvc-complexity/)
Uses several metrics to compare the complexity of TodoMVC as implemented by various frameworks (jQuery, AngularJS, Ember.js, Backbone.js, etc). This can provide a way to see which frameworks encourage the use of simpler constructs.

## [Filtering Arrays with Array#filter](http://designpepper.com/blog/drips/filtering-arrays-with-array-filter)
Josh Clanton explains how this built-in array method works.

## [Write Your CSS with&nbsp;JavaScript](http://davidwalsh.name/write-css-javascript)
David shows off how to use ‘absurd.js’, an intriguing library that allows you to write stylesheets dynamically using JavaScript.

## [Creating a Realistic Rain Effect with Canvas and JavaScript](http://flippinawesome.org/2013/09/23/creating-a-realistic-rain-effect-with-canvas-and-javascript/)
Marek Brodziak show the techniques used to build RainyDay.js, a library creating a realistic looking rain effect on HTML canvas.

## [Unicode and JavaScript](http://www.2ality.com/2013/09/javascript-unicode.html)
Axel Rauschmayer first gives a brief but thorough overview of Unicode and then explains how to use it properly in JavaScript.

## [JavaScript Y Combinator](http://buzzdecafe.github.io/code/2013/09/05/javascript-y-combinator/)
Michael Hurley uses JavaScript code to explain the [Y combinator](http://en.wikipedia.org/wiki/Fixed-point_combinator#Y_combinator), a fixed-point combinator in lambda calculus.

## [Introducing Responsive Web Typography With FlowType.JS](http://coding.smashingmagazine.com/2013/09/18/introducing-flowtype-js/)
A jQuery plugin that adapts font sizes to the size of the device a web page is displayed in (similar to how responsive design already works for images).

## [ServiceWorkers Explained](https://github.com/slightlyoff/ServiceWorker/blob/master/explainer.md)
The HTML5 Application Cache helps web apps to operate offline. Alas, this API has [well-known limitations](http://alistapart.com/article/application-cache-is-a-douchebag). Therefore, Alex Russell proposes the ServiceWorkers API, which works around those limitations by giving developers lower-level primitives.

## [Getting Started with Web Components and Polymer.js](http://4waisenkinder.de/blog/2013/09/21/getting-started-with-web-components-and-polymer-dot-js/)

## [Automating Complex Workflows with Grunt Custom Tasks](http://flippinawesome.org/2013/09/23/automating-complex-workflows-with-grunt-custom-tasks/)
Ben Farrell explains to create your own Grunt custom tasks that can be used to automate complex workflows, in his example he is extracting audio from video.

## [How to Create a Prototype Angular UI without a Back-end](http://www.opensourceconnections.com/2013/09/16/prototype-angular-uis-without-a-backend/)

## [Playing with Chaos: Programming Fractals and Strange Attractors in JavaScript](http://www.amazon.com/dp/B00FA9CX2Y)
A Kindle book by Keith Peters.

## [Backbone.js Routers](http://www.sitepoint.com/backbone-js-routers/)
Backbone.js routers are used to keep track of state in your app. Routers translate a path in the URL or in a hash of the URL into application state. In other words, routers map a path to a function. Andy Appleton shows off how to use them in this 8 minute screencast.
